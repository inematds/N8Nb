Por Que Contexto Hibrido?

Em um RAG tipico, voce busca os chunks mais relevantes semanticamente. Mas e se o contexto ao redor do chunk for importante para entender a resposta?

RAG Simples

Retorna apenas o chunk relevante. Pode perder contexto crucial que estava no paragrafo anterior ou seguinte.

Hybrid Context

Retorna o chunk + 2 linhas antes + 2 linhas depois. Mantem a narrativa e relacionamentos entre conceitos.

Fluxo de Ingestao e Vetorizacao

Monitoramento Google Drive

Observa pasta para novos documentos e extrai conteudo de texto automaticamente.

Geracao de Embeddings OpenAI

Converte chunks de documento em vetores usando modelo text-embedding-large.

Armazenamento Supabase com Indexacao

Faz upload dos chunks embeddados com indexacao de linha para retrieval sequencial.

Fluxo de Retrieval Hibrido

Trigger de Chat

Recebe queries via chat nativo do N8N (configuravel para webhook/apps de mensagem).

Execucao de Busca Semantica

Agente invoca vector content tool (sub-workflow) para consultar Supabase por chunks relevantes.

Expansao de Contexto

JavaScript extrai 2 linhas antes e 2 depois de cada chunk recuperado para contexto circundante.

Agregacao de Contexto

Combina chunks semanticamente relevantes + texto vizinho em bloco de contexto unificado.

Processamento Gemini

Alimenta contexto expandido para modelo Gemini (escolhido por janela de contexto maior).

Modelo Fallback

Alterna automaticamente para modelo backup se primario der timeout ou bater limite de tokens.

Proposta de Valor

Maior Precisao nas Respostas

Chunks contextuais "antes e depois" reduzem interpretacao errada de fragmentos de texto isolados.

Melhor Entendimento Narrativo

Mantem fluxo do documento e relacionamentos entre conceitos que busca semantica pura perde.

Fallback Inteligente

Alterna modelos automaticamente para prevenir falhas de workflow por janelas de contexto muito grandes.

Controle Flexivel de Contexto

Ajuste profundidade de retrieval (linhas antes/depois) baseado em tradeoffs de precisao vs. custo.

Configuracao Importante

1. Sub-Workflow Separado

Copie o sub-workflow "vector content tool" para um workflow separado no N8N. Certifique-se que o nome corresponde ao que o agente procura.

2. Ajuste o Contexto

Por padrao: busca 4 chunks + 2 linhas antes e depois de cada = ~20 linhas totais de contexto. Ajuste conforme necessidade.

3. Use Gemini

Gemini foi escolhido pela janela de contexto maior. Configure o modelo fallback para evitar erros com contextos muito grandes.

4. Teste Cuidadosamente

Teste para garantir que nao esta repetindo muito contexto e que o error handling esta funcionando corretamente.
